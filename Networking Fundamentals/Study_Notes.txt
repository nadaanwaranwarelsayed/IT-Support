**Connection**
Copper cross-over  --> connecting between two devices of the same type (pc,router)
Copper straight-through --> connecting between two different devices (PC to Switch) or (Router to Switch)
Console cable --> connecting between pc , new router/new switch to program it (RS232 port)
Single fiber --> connection between countries , thin , expensive
Multi fiber --> connection between local devices , thick , less expensive than single
FastEthernet0 --> the network port in devices 

**Notes**
IP address  --> logical address used to identify a device on a network.
Subnet mask --> defines which part of the IP address belongs to the Network and which part belongs to the host (the device). It determines the size/range of the network
server belongs to end devices
devices request IPs from server so the server must be have Static IP 
DNS (domain name system) -->  translates human-readable domain names into machine-readable IP addresses to facilitate network communication
Router --> Connects two different networks, enabling all PCs across those networks to communicate with each other ,
           Each port on the router acts as a Default Gateway for its connected network 

in dynamic routing :
RIPv2 --> protocol depends on hops (number of ways between router , router)
     max number of hops = 15 if hops were > 15 the router consider it lost 
OSPF --> protocol depends on the speed (cost) 
         if the connector was fiber type (high speed) the cost wil be 1 (very little)
                              old type (low speed)                  100 (very high)
the 
OSPF always choose the high speed way , RIPv2 always choose the little hops way even if it less speed
